# PhotoFold Technical Contracts

**Status:** Gate 1 measured contract plus Phase 4P.1 generalized-folding extension
**Authority:** `PhotoFold_Developer_PRD.md`, clarified by `IMPLEMENTATION_PLAN.md`

This document fixes the terms used by the implemented Gate 1 experiment. Measured results remain external artifacts generated by the benchmark; they are not hard-coded into the application.

## Measurement contract

### Source bytes

```text
original_total_bytes = sum(stat(exact uploaded source file))
```

Source bytes are measured before EXIF normalization, decoding, resizing, or transcoding.

### Package bytes

```text
package_total_bytes = stat(closed final .photofold archive)
byte_delta = original_total_bytes - package_total_bytes
percent_change = byte_delta / original_total_bytes * 100
bytes_saved = max(byte_delta, 0)
percent_saved = max(percent_change, 0)
```

The product package count includes ZIP overhead and every file in the downloadable archive. Inspection outputs outside the archive do not count. Metrics inside the archive omit `package_total_bytes` to avoid a self-referential file-size calculation; the external run result owns that value.

### Quality

- Normalize EXIF orientation before quality comparison.
- Define the normalized display-oriented dimensions as output dimensions.
- Reopen reconstructions through the public package decoder.
- Calculate RGB SSIM with `data_range=255` and `channel_axis=2` for opaque photos.
- Record per-frame, mean, and minimum SSIM.
- Quality thresholds may be committed only after a real rate-distortion sweep and human visual review.
- For the `hdrplus-static` Gate 1 run, the measured/visually reviewed thresholds are mean SSIM ≥ 0.85 and every frame SSIM ≥ 0.82. They are dataset-specific, not product guarantees.
- Difference heatmaps and a local-error diagnostic supplement SSIM; they do not replace it.

## Transform contract

- Store a row-major 3×3 matrix named `reference_to_target`.
- Coordinates are normalized, display-oriented full-resolution pixel coordinates.
- The matrix maps the decoded reference base into the target output canvas.
- Record transform type, interpolation mode, border mode, output width, and output height.
- Store every changed-region patch, lossless mask, and bounding box in target-frame coordinates.
- Pixels outside valid warped-base coverage are changed pixels.

### Resolution-independent alignment evidence

- `median_reprojection_error` is measured in `analysis_pixels` on the EXIF-normalized analysis copy whose longest dimension is at most 800 pixels.
- The error is not divided by the analysis scale or converted back to source pixels. The same scene therefore receives materially equivalent alignment evidence at different capture resolutions.
- The committed Phase 4P.1 decision records `analysis_max_dimension = 800`, `max_median_reprojection_error = 2.0 analysis_pixels`, and `min_inlier_ratio = 0.80` in every prototype analysis result. These remain measured prototype settings rather than general product guarantees.
- Full-resolution `reference_to_target` matrices are still serialized in normalized display-oriented source coordinates. Changing the error unit does not change the package transform convention.
- A frame joins the shared group only when its selected transform satisfies both measured alignment thresholds. A failed measurement, low inlier ratio, or differing normalized dimensions selects independent fallback for that frame; it does not weaken or bypass the thresholds.

## Package schema

The normative generated schema is `packages/contracts/photofold-manifest.schema.json`; its TypeScript counterpart is generated at `packages/contracts/src/photofold-manifest.ts`.

Version 0.2 archive layout:

```text
moment.photofold
├── manifest.json
├── base.webp
├── frames/{index}/frame.json
├── frames/{index}/patches/{patch}.webp
├── frames/{index}/patches/{patch}-mask.png
├── frames/{index}/source.{jpg|png|webp}  # independent fallback only
└── metadata/
    ├── analysis.json
    └── metrics.json
```

Each frame declares exactly one storage mode:

- `shared_reference`: the one identity-transform frame encoded by `base.webp`;
- `shared_delta`: a frame reconstructed from the base, its full-resolution transform, and zero or more target-space patches/masks; or
- `independent_source`: the exact uploaded JPEG, PNG, or WebP payload, decoded and EXIF-normalized by the public package decoder.

Every frame records its normalized width and height. `shared_scene` packages contain only shared modes, `hybrid` packages contain shared and independent modes, and `independent_only` packages have a null base/reference and contain only independent sources. A useful shared group contains at least two frames; otherwise the package is independent-only.

The manifest inventories the byte size and SHA-256 checksum of every referenced non-manifest asset. It cannot checksum itself. The inventory must exactly equal the storage-mode-derived allow-list; unreferenced assets, missing frame metadata, frame/manifest disagreement, unsafe paths, dimension mismatches, codec mismatches, and checksum failures are rejected. The external run result may contain the final archive checksum.

The public decoder reconstructs an independent member using only its closed-package payload: decode the declared format, apply EXIF orientation, reject unsupported transparency or unsafe pixel counts, convert the normalized opaque image to RGB, and verify its per-frame dimensions. This produces the exact normalized source pixels without a transcode. Version 0.1 Gate 1/Gate 3P packages are migrated in memory and remain decodable; they are never rewritten.

## Suitability and region definitions

- Every upload retains its original index and an explicit accepted/rejected disposition.
- Folding requires 5–20 accepted frames.
- Gate 1's primary dataset must accept and reconstruct every input frame.
- `split_recommended` is advisory only.
- Phase 4P.1 stores differing normalized dimensions independently without resizing, cropping, or distorting them.

```text
changed_region_percent = changed_mask_pixels / output_pixels * 100
shared_region_percent = valid_warped_base_pixels_not_changed / output_pixels * 100
```

Aggregates are weighted by output pixel count. Analysis-resolution values are labeled estimates; final values come from full-resolution masks.

Successful validation always selects one of `shared_scene`, `hybrid`, or `independent_only`. Analysis records shared/fallback counts, ordered per-frame dispositions, measured alignment evidence, and an explicit reason for every fallback. Folding remains enabled for all three strategies. Fallback warnings state that storage savings may be reduced or eliminated.

Terminal evaluation remains evidence-based:

- `complete`: package integrity and quality pass, and the closed archive is smaller than exact uploads;
- `complete_no_savings`: integrity and quality pass, but the archive is not smaller;
- `failed_quality`: package-only reconstruction succeeds, but the recorded quality thresholds fail; and
- `failed`: validation, package construction, integrity, or reconstruction cannot complete.

## Error taxonomy

Every later processing error uses a stable code, user-safe message, stage, affected frame indices, retryability, and development-only debug detail.

| Code | Stage | Meaning |
|---|---|---|
| `INVALID_FILE_COUNT` | upload | Fewer than 5 or more than 20 files. |
| `UNSUPPORTED_FILE_TYPE` | upload | File format is not supported. |
| `IMAGE_DECODE_FAILED` | preprocess | Image cannot be decoded safely. |
| `DIMENSIONS_INCOMPATIBLE` | preprocess | Legacy/reserved compatibility error; Phase 4P.1 valid mixed dimensions use independent fallback. |
| `CHECKSUM_MISMATCH` | validation | Curated/source asset does not match its manifest. |
| `INSUFFICIENT_OVERLAP` | analyze | Shared scene overlap is below the configured threshold. |
| `INSUFFICIENT_FEATURES` | align | Too few reliable feature matches. |
| `IMPLAUSIBLE_TRANSFORM` | align | Transform geometry fails sanity checks. |
| `ALIGNMENT_FAILED` | align | A frame cannot be aligned reliably. |
| `PACKAGE_VALIDATION_FAILED` | package | Schema, path, checksum, codec, or asset validation fails. |
| `RECONSTRUCTION_FAILED` | reconstruct | Package-only reconstruction cannot finish. |
| `QUALITY_BELOW_THRESHOLD` | evaluate | A completed reconstruction misses the recorded threshold. |
| `NO_MEANINGFUL_SAVINGS` | evaluate | The closed archive is not smaller than exact sources. |
| `JOB_BUSY` | service | The single-worker processor cannot accept another fold. |
| `MOMENT_NOT_FOUND` | service | Moment does not exist or has expired. |
| `SEMANTIC_PROVIDER_FAILED` | semantic | Optional provider failed; deterministic processing continues. |

Dataset/checksum and health concerns remain the only HTTP behavior. Gate 1 implements package validation, reconstruction, quality evaluation, and export through the CLI; later service errors remain contract reservations rather than implemented API routes.

## Independent-WebP control

Gate 1 encodes every normalized frame independently at every integer WebP quality from 1 through 100. This exhaustive encoder-setting curve is deliberately denser than the PhotoFold parameter sweep and prevents a coarse control grid from overstating relational savings.

PhotoFold proves relational benefit only when:

```text
package_total_bytes < independent_webp_total_bytes
```

at equal or better mean and minimum SSIM. Beating only the uploaded source bytes is a storage reduction, not proof that shared-scene encoding caused it.

The normative result shape is `packages/contracts/experiment-result.schema.json`. It distinguishes `storage_reduction_pass` from `relational_hypothesis_pass` and never allows the latter without the former's measured evidence.

## Gate 1 selected configuration

The real sweep selected WebP quality 70 for the base and patches, difference threshold 24, dilation radius 8, decoder-side feather radius 2, minimum component area 1,152 pixels, and maximum patch tile 384×384. Package JSON uses compact serialization because the image archive is `ZIP_STORED`. Patch crops retain real target pixels outside their binary masks so lossy WebP cannot bleed zero-fill into composited edges; the mask alone controls application. `configs/gate1.yaml` is the executable source of these values, and `benchmark.json` records them and the config checksum for every run.
